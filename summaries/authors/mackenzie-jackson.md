# Topic: Author: Mackenzie Jackson

## Executive Summary

The two records attributed to Mackenzie Jackson are both BSidesLV 2025 sessions focused on software supply chain security, but they approach the topic from complementary angles. One is a hands-on workshop about building a pragmatic open-source security stack across code, CI/CD, application testing, secrets detection, infrastructure-as-code, and cloud workflows [record_id:2440]. The other is a research-oriented talk about using large language models to detect threats and vulnerabilities in open-source software ecosystems, including malicious packages, undisclosed vulnerabilities, and an alleged live Lazarus Group operation affecting the Ripple/XRP cryptocurrency SDK [record_id:2474].

Collectively, these records position Jackson’s work around practical, scalable defense for the open-source software supply chain. Recurring concerns include dependency risk, leaked secrets, malicious packages, attacker abuse of public package ecosystems, and the need to detect problems earlier in development and release pipelines [record_id:2440] [record_id:2474]. The records also show a dual emphasis: accessible tooling for ordinary teams that cannot afford expensive enterprise security platforms, and advanced research methods using frontier LLMs to analyze public software ecosystem data at scale [record_id:2440] [record_id:2474].

The evidence base is small—only two BSidesLV 2025 abstracts—so it supports a focused but limited characterization. ## Major Themes And Trends

A central theme across both records is that the open-source software supply chain is a major security battleground. In the workshop, Jackson frames practical security around common supply chain and application risks such as Log4Shell, dependency poisoning, and leaked secrets [record_id:2440]. In the research talk, the supply chain appears as an ecosystem-level attack surface where adversaries publish malicious packages, exploit trust in public registries, and attempt to deliver or persist payloads undetected [record_id:2474]. The shared concern is that attackers increasingly target the development and distribution pathways that connect code, dependencies, build pipelines, and downstream consumers.

A second theme is democratizing security through tools and workflows that ordinary teams can actually adopt. The workshop explicitly contrasts expensive commercial security products and “AI-fueled promises” with “tools that work” and “won’t wreck the budget” [record_id:2440]. It emphasizes battle-tested open-source tools and integration into existing developer workflows through CI/CD, IDEs, and pre-commit hooks [record_id:2440]. This suggests a practitioner-centered philosophy: security controls should be embedded early, automated where possible, and affordable enough for teams without large enterprise budgets.

A third theme is early detection. The workshop says it will show how to detect and stop real attack paths early, including Log4Shell, dependency poisoning, and leaked secrets [record_id:2440]. The LLM-focused talk similarly emphasizes identifying vulnerabilities and malicious behavior in large public datasets before harm propagates, including the claim that the research prevented a backdoor from being shipped in the official Ripple/XRP cryptocurrency SDK [record_id:2474]. Across both records, the goal is not only response after compromise but earlier intervention in the lifecycle of code, packages, and releases.

A fourth theme is scaling security analysis. In the workshop, scale is addressed operationally: integrate tools across CI/CD, IDEs, and pre-commit hooks so defenses become part of routine engineering practice [record_id:2440]. In the LLM talk, scale is addressed analytically: use frontier LLMs such as GPT-4 to analyze large public datasets including changelogs, package metadata, and behavioral signals, then augment vulnerability discovery and threat triage [record_id:2474]. The two approaches are complementary: one scales adoption through workflow integration, while the other scales detection through automated reasoning over ecosystem data.

A fifth theme is the tension between open-source trust and open-source risk. The workshop defends open-source tools as capable building blocks for security programs [record_id:2440]. The research talk, however, focuses on how attackers abuse trust in open-source ecosystems to gain access, deliver payloads, and persist undetected [record_id:2474]. Rather than contradicting each other, these records present a nuanced view: open source is both a defensive advantage and an adversarial target. Jackson’s contribution, as represented here, is to argue for securing the open-source ecosystem using both practical open-source controls and advanced analytic techniques.

The workshop also highlights integration patterns rather than standalone scanning. Jackson’s stated blueprint includes using CI/CD, IDEs, and pre-commit hooks [record_id:2440]. This indicates an emphasis on shifting security checks into developer workflows and automated build/release processes. Pre-commit hooks catch issues before code is committed; IDE integrations provide immediate developer feedback; CI/CD enforcement supports repeatable checks before deployment or release. The abstract frames these as practical ways to detect and stop attack paths early [record_id:2440].

The second record discusses LLM-assisted detection across open-source software supply chain datasets [record_id:2474]. The stated inputs include changelogs, package metadata, and behavioral signals [record_id:2474]. The approach appears to use LLMs not as generic chatbots but as analytic aids for identifying suspicious patterns, undisclosed vulnerabilities, and attacker behavior at scale. The LLM talk also introduces the “open-source kill chain” concept [record_id:2474]. This is a conceptual method rather than a single tool: it provides a way to reason about attacker stages within open-source package ecosystems. It may help downstream researchers compare package-publishing attacks, dependency confusion, maintainer compromise, malicious update campaigns, and other public-registry threats under a shared model, though the abstract itself does not enumerate the full model.

Across both records, the methods emphasize reproducibility and operational relevance. The workshop promises live demos and a practical integration blueprint [record_id:2440]. The research talk says it is grounded in operational examples, focused on reproducible techniques, and provides a current view into how APTs and malware authors exploit open-source ecosystems [record_id:2474]. The common approach is applied rather than purely theoretical.
