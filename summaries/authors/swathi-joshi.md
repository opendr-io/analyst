## Major Themes And Trends

### From Cloud Security To AI-Native Security Challenges

A central theme across both records is that cloud security is no longer the only major source of acceleration and complexity for security teams. Record 2259 states that “AI is reshaping security faster than cloud ever did,” placing AI at the center of a new shift in security practice [record_id:2259]. Record 2282 similarly notes that “cloud security is moving faster than ever” while also emphasizing that “ML for security and security for ML are more crucial than ever” [record_id:2282].

Together, these records suggest a view of the security landscape in which cloud, AI, and ML are converging. The concern is not simply how to secure cloud infrastructure or how to secure AI models in isolation, but how to build security programs capable of operating in environments where cloud-native systems, AI capabilities, and ML-enabled security tooling are intertwined.

### Security Program Modernization Without Hype

Another major theme is the tension between modernization and hype. Record 2282 is explicit that practitioners are “bogged down by the constant chase of the latest attacks and shiny tech objects” and promises “no hot takes,” instead focusing on “good ol’ first-principle approaches” [record_id:2282]. This establishes a critical posture toward reactive or fashion-driven security.

Record 2259, by contrast, is framed around emerging AI security needs: “real-world threat models,” “AI-first SOC teams,” and “the gaps legacy tools can't fill” [record_id:2259]. But when read alongside record 2282, the panel topic does not appear to celebrate novelty for its own sake. Rather, the combined corpus suggests that Joshi’s BSidesSF 2025 contribution involves adapting to AI-driven change while grounding that adaptation in durable security principles.

This pairing is important: one record foregrounds new AI/cloud realities; the other cautions against chasing novelty. The synthesis is a pragmatic modernization agenda—security teams must address AI and ML risks, but they should do so through disciplined principles rather than vendor-driven or trend-driven responses.
