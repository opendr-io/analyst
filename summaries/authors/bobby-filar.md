# Topic: Author: Bobby Filar

## Executive Summary

The two records attributed to Bobby Filar are CAMLIS talks from 2018 and 2019, both centered on security applications of machine learning and both oriented toward practical defensive analysis rather than purely academic model development. The records show a consistent interest in where machine learning systems succeed or fail in malware and endpoint-security contexts: one talk examines the vulnerability and interpretability of tree-based malware classifiers, while the other introduces a graph-based framework for detecting anomalous endpoint process relationships and attack sequences.

Across the records, Filar’s work emphasizes context, model introspection, and defensive utility. In the 2018 “TreeHuggr” talk, the focus is on understanding where gradient-boosted decision trees and random forests are brittle or susceptible to adversarial evasion, especially in malware detection settings using the EMBER dataset and model [record_id:249]. In the 2019 “ProblemChild” talk, the focus shifts from static file-level classification to behavioral endpoint detection, arguing that parent-child process context can reveal malicious activity that may look normal when observed in isolation [record_id:221].

Taken together, the records portray Filar as working at the intersection of adversarial machine learning, malware detection, endpoint behavior analytics, and interpretability. The evidence base is small—only two records—but coherent: both talks frame machine learning not as a black-box replacement for security expertise, but as a tool for surfacing weak points, anomalous structures, and higher-order relationships that can improve defensive detection and analysis.

## Research Landscape

The included records are both conference talk abstracts from CAMLIS, one from 2018 and one from 2019. They are not full papers, code repositories, slide decks, or empirical results documents, so the available evidence primarily describes proposed systems, motivations, and intended demonstrations rather than providing detailed technical validation. The two records are nevertheless substantial abstracts that identify the problem domains, proposed approaches, datasets or evaluations, and intended security relevance.

The 2018 record, “TreeHuggr: Discovering where tree-based classifiers are vulnerable to adversarial attack,” sits in the area of machine learning model security and adversarial ML for malware detection. It discusses tree-based classifiers, including gradient-boosted decision trees and random forests, and frames them as high-performing models in information security tasks such as malware detection [record_id:249]. The abstract contrasts the abundance of adversarial methods against deep learning with the relative scarcity of attacks or analysis methods for non-differentiable tree ensembles [record_id:249]. It proposes TreeHuggr as a way to analyze split points within ensemble trees to identify areas of susceptibility, brittleness, conflicting evidence, and interpretability gaps [record_id:249].

The 2019 record, “ProblemChild: Discovering Anomalous Patterns based on Parent-Child Process Relationships,” belongs more directly to detection engineering, endpoint detection and response, threat hunting, and behavioral analytics. It argues that malware attacks are increasingly not reducible to standalone executables or scripts, but instead exhibit “conspicuous process heritage” that may be ignored by models relying only on static features such as PE header metadata [record_id:221]. The talk introduces ProblemChild as a graph-based framework that uses process relationships, machine learning, weighted graphs, community identification, and conditional process probabilities to detect larger attack sequences and identify anomalous or suppressible process-level events [record_id:221].

The overall research landscape represented by these records is therefore narrow but thematically connected. Both talks deal with malware and security analytics. Both recognize limitations in common machine-learning approaches. Both introduce named systems—TreeHuggr and ProblemChild—intended to expose hidden structure: TreeHuggr exposes fragile regions and blind spots in classifier decision spaces, while ProblemChild exposes anomalous process-chain communities and parent-child process patterns in endpoint telemetry [record_id:249] [record_id:221].

## Major Themes And Trends

A central theme across the records is the inadequacy of looking at security artifacts in isolation. In TreeHuggr, the isolated view is a classifier’s surface-level performance: tree-based classifiers may achieve state-of-the-art results in malware detection, but that does not mean analysts understand where they are vulnerable or brittle [record_id:249]. In ProblemChild, the isolated view is a standalone executable, script, or static feature vector: malware behavior may appear normal when considered as an individual process or file, but become suspicious when analyzed through parent-child process context [record_id:221].

Another recurring theme is defensive interpretability. TreeHuggr is explicitly framed as a way to understand “where” a model is most susceptible to evasion, not merely as a method for generating an adversarial example from a starting point [record_id:249]. The abstract ties this to understanding a model’s blind spots, identifying insufficient or conflicting class evidence, and improving interpretability [record_id:249]. ProblemChild similarly uses structure to make detection more intelligible: by constructing weighted graphs and identifying communities of events, it aims to group seemingly disparate events into larger attack sequences [record_id:221]. While the 2019 abstract is less explicitly about model interpretability, it is still concerned with making attack behavior visible by modeling relationships among processes.

A third theme is the use of machine learning as an aid to security reasoning rather than a stand-alone oracle. TreeHuggr does not present adversarial ML simply as an offensive evasion mechanism. Instead, it emphasizes “defensive uses of adversarial research” and argues that understanding attack susceptibility can produce insights into model interpretability [record_id:249]. ProblemChild likewise positions ML as part of a detection workflow: it uses machine learning to derive a weighted graph, but also incorporates conditional probabilities of child processes given parents to elevate or suppress anomalous communities [record_id:221]. This suggests a hybrid analytic posture in which statistical signals, graph structure, and security domain knowledge interact.

There is also a progression from static malware-classification concerns in 2018 toward behavioral endpoint detection in 2019. The TreeHuggr talk is grounded in classifiers and feature spaces, with a demonstration against the open-source EMBER dataset and malware model [record_id:249]. The ProblemChild talk addresses process chains, living-off-the-land techniques, persistence, defense bypasses, execution actions, and MITRE ATT&CK evaluation replay [record_id:221]. This shift reflects a broader security trend: defenders increasingly recognize that modern attacks may evade static detection, use legitimate tools, and require behavioral context to identify. The ProblemChild abstract states this directly by noting that living-off-the-land activity can appear normal in isolation but suspicious in parent-child context [record_id:221].

The records also share a concern with false positives, blind spots, and practical evaluation. TreeHuggr seeks locations in feature space where evidence is insufficient or conflicting, implying that model errors and ambiguous regions deserve focused study [record_id:249]. ProblemChild specifically promises to highlight detections and false positives observed during a replay of the 2018 MITRE ATT&CK evaluation for APT3 [record_id:221]. Both records therefore treat defensive ML systems as imperfect and emphasize methods for understanding, evaluating, and improving them.

## Methods, Tools, And Approaches Discussed

The first named approach is TreeHuggr, described as an algorithm for analyzing split points in each tree of an ensemble classifier [record_id:249]. The models in scope include gradient-boosted decision trees and random forests, which the record says provide state-of-the-art performance in many information security tasks, including malware detection [record_id:249]. TreeHuggr’s method is not presented as a gradient-based adversarial attack; rather, it responds to the non-differentiability of tree-based models by examining tree split points to infer where the classifier may be susceptible to evasion [record_id:249].

TreeHuggr’s analytic target is the structure of the feature space. The abstract describes looking for regions where there is insufficient or conflicting evidence for a class label, as well as regions where a decision boundary is “wrinkled” [record_id:249]. These descriptions suggest a method focused on identifying unstable or ambiguous boundary areas inside tree ensembles. The stated goal is to understand the attack space, model blind spots, and interpretability, with an emphasis on “where the model is most susceptible” rather than producing a specific evasion path from a chosen starting sample [record_id:249]. The record says the talk would provide an example-driven demonstration against the open-source EMBER dataset and malware model [record_id:249].

The second named approach is ProblemChild, described as a graph-based framework for discovering malicious software based on process relationships [record_id:221]. Its core premise is that process heritage—especially parent-child process relationships—contains meaningful detection context often missed by machine-learning models that rely only on static features such as PE header metadata [record_id:221]. The approach applies machine learning to derive a weighted graph, then uses that graph to identify communities of seemingly disparate events and assemble them into larger attack sequences [record_id:221].

ProblemChild also uses conditional probability, specifically `P(child | parent)`, to automatically uncover rare or common process-level events [record_id:221]. The record states that these probabilities can be used to elevate or suppress anomalous communities [record_id:221]. This is important because it indicates a calibration mechanism: not every unusual-looking community should necessarily be treated as malicious, and not every rare relationship is equally meaningful. The approach combines graph analytics, probabilistic context, and machine learning to reason about endpoint behavior.

The ProblemChild abstract also situates the method in relation to common security-vendor heuristics or “analytics” used to identify process events [record_id:221]. It notes that attacker techniques can be chained to perform persistence, defense bypasses, and execution actions [record_id:221]. In this context, ProblemChild appears to be positioned as a way to discover or support analytics based on process relationships, rather than relying solely on manually authored rules. Its evaluation context is a replay of the 2018 MITRE ATT&CK evaluation for APT3, with attention to both detections and false positives [record_id:221].

Together, TreeHuggr and ProblemChild illustrate two complementary approaches. TreeHuggr is inward-looking: it probes a model’s decision structure to expose fragility and interpretability issues [record_id:249]. ProblemChild is outward-looking: it models relationships among endpoint events to expose attacker behavior that static or isolated features may miss [record_id:221]. Both approaches are designed to help defenders understand where machine-learning-based security systems can be improved.

## Notable Talks, Records, And Evidence

The 2018 CAMLIS talk “TreeHuggr: Discovering where tree-based classifiers are vulnerable to adversarial attack” is notable because it focuses on a comparatively underexplored adversarial ML target: tree-based classifiers used in security [record_id:249]. The abstract states that adversarial methods for evading deep learning classifiers are abundant, while less research has attacked tree-based classifiers because non-differentiability significantly increases attack cost [record_id:249]. This framing is important because it identifies a gap in adversarial ML research and positions TreeHuggr as a defensive analysis tool rather than a simple attack generator.

The key contribution described in the TreeHuggr record is its focus on locating susceptibility within an ensemble model. By analyzing split points and identifying weak or ambiguous regions of feature space, TreeHuggr aims to reveal where a tree-based malware model may be brittle [record_id:249]. The record’s emphasis on the EMBER dataset and malware model also matters: EMBER is a known open-source malware-classification benchmark, so the talk is presented as grounded in a reproducible or at least publicly recognizable evaluation setting [record_id:249]. However, the abstract does not provide detailed results, metrics, or examples, so downstream researchers should treat the record as evidence of the talk’s claims and design goals, not as proof of performance.

The 2019 CAMLIS talk “ProblemChild: Discovering Anomalous Patterns based on Parent-Child Process Relationships” is notable because it moves from model vulnerability analysis to endpoint behavioral detection [record_id:221]. It argues that malware attacks often have process heritage that static-feature ML models ignore, and that living-off-the-land techniques become more suspicious when examined in parent-child context [record_id:221]. This record is representative of a detection-engineering trend toward modeling sequences, chains, and relationships rather than isolated events.

ProblemChild’s described contribution is a graph-based framework that applies machine learning to derive a weighted graph and identify communities of events as larger attack sequences [record_id:221]. Its use of `P(child | parent)` to elevate or suppress anomalous communities is also a notable feature because it suggests an approach for distinguishing rare-but-benign from rare-and-suspicious process relationships [record_id:221]. The record’s reference to a replay of the 2018 MITRE ATT&CK evaluation for APT3 provides a concrete evaluation scenario and ties the work to a recognized adversary-emulation benchmark [record_id:221]. As with TreeHuggr, however, the record does not include detailed quantitative outcomes; it only states that detections and false positives observed during the evaluation would be highlighted [record_id:221].

As a pair, the records matter because they show continuity in Filar’s work: both are about using machine learning to make security systems more explainable, robust, and behaviorally aware. TreeHuggr addresses how defenders can identify vulnerable regions in a classifier before or alongside adversarial evasion concerns [record_id:249]. ProblemChild addresses how defenders can uncover malicious process sequences that evade models focused on static, standalone indicators [record_id:221].
