# Topic: Source: blackhat (2026)

## Executive Summary

## Major Themes And Trends

### 1. Agentic AI turns intent, delegation, and workflow state into security boundaries

A major shift across the records is that AI systems are no longer treated as passive text generators. They are described as systems that navigate browsers, call tools, execute code, read files, manage workflows, and act with delegated authority. This creates new security boundaries around intent, state, tool routing, memory, and orchestration.

AI browsers are a focal point. One record claims that every popular AI-powered browser analyzed was vulnerable to indirect prompt injection that could lead to data exfiltration or account takeover, using techniques ranging from hidden HTML instructions to image steganography [record_id:2583]. Another argues that agentic browsers intentionally dismantle long-standing isolation assumptions such as Same-Origin Policy, localhost restrictions, filesystem isolation, and script execution boundaries, enabling “PleaseFix” and “Intent Collision” attacks that result in zero-click account takeover, local exfiltration, persistence, and even local RCE [record_id:2667]. These two records together suggest a trend: browsers are being redesigned around agency before the industry has stable models for constraining that agency.

AI assistant and workflow records make the same point outside browsers. A Copilot-focused talk introduces “Remote Prompt Execution,” chaining safety-filter bypass, sandbox privilege escalation, and an Azure Container Registry Image Streaming flaw to compromise a user’s Copilot session and access Microsoft 365 data [record_id:2606]. A retail assistant talk describes indirect prompt injection through product comparison and differential handling of search parameters, leading to internal prompt disclosure, tool syntax recovery, backend tool-code execution, and exposed API keys [record_id:2641]. A record on official AI workflows reports trust-handoff failures across Claude Code, Gemini CLI, and Codex, where state marked safe by one stage is later interpreted more powerfully, leading to shell execution, secret exposure, persistence, or policy bypass [record_id:2644]. Another record generalizes this problem to agent frameworks such as LangChain, LangGraph, CrewAI, AutoGen, Microsoft Agent Framework, and Google ADK, where prompt-controlled content crosses from data plane into orchestration, memory, routing, system instructions, or native parsing [record_id:2648].

Several records frame the problem conceptually. One argues that current AI regulations fail because they assume software executes instructions, while agentic systems form plans and act with “intent”; it proposes intent monitoring, agent identity standards, and behavioral audits [record_id:2597]. Another says agentic security is hard because it violates hidden assumptions about control, trust, authority, intent, and feedback, and calls for “governing agency” rather than merely “securing agents” [record_id:2621]. A threat-modeling record introduces PHANTOM-B as a practical mnemonic for LLM product teams, positioned as faster and more deployment-oriented than broad frameworks such as MITRE ATLAS or NIST AI RMF [record_id:2661].

Defensive responses are emerging but incomplete. Roblox’s Claude Code sandboxing record describes multi-layer sandboxes for macOS, Linux, Windows, and cloud VMs, with an ML Gateway, managed system prompts, and VPN profiles, motivated by an internal test in which hidden GitHub Issue instructions convinced Claude Code to upload credentials to a public repository [record_id:2655]. GAVEL proposes rule-based detection over neural activations rather than surface text, using “Cognitive Elements” and shareable rules analogous to Snort/YARA [record_id:2652]. A policy panel and AI-security policy talks address staged release, public-private coordination, frontier model access, infrastructure protection, and model-weight exfiltration [record_id:2636] [record_id:2686] [record_id:2694].

### 2. AI is becoming an exploit-development accelerator, but bounded workflows outperform “magic agents”

A second major trend is AI used to discover, rank, reproduce, exploit, classify, or mitigate vulnerabilities. The tone is not uniformly hype-driven. Several records emphasize bounded, tool-assisted, inspectable workflows rather than unconstrained autonomy.

James Kettle’s HTTP Terminator talk asks whether an autonomous system can invent new attack techniques and use them at scale. The abstract claims the system discovered new HTTP desync triggers, gadgets, and exploits against banks, security products, and government infrastructure, while also identifying areas where human/AI loops outperform autonomy and some discoveries remain beyond AI [record_id:2594]. Prompt2Own similarly reports an LLM kernel agent assisting real kernel exploit development across Linux, Android, and Qualcomm DSP cases, including generating PoCs and finding a previously unknown Linux crypto subsystem use-after-free, while evaluating model contributions by exploit-development phase [record_id:2612].

Records on code and firmware analysis are especially concrete. SiftRank reframes binary vulnerability discovery as information retrieval, using LLMs to rank batches of decompiled functions; on BinPool it claims 2.26x greater precision than zero-shot classification and reports a commercial network power controller case where a hidden diagnostic endpoint ranked first led to RCE and CVE-2026-41446 [record_id:2619]. Another IoT firmware talk proposes a semi-automatic methodology using frontier LLMs with decompilers and debugging tools under researcher-defined boundaries, claiming 30 vulnerabilities in five days across selected targets, including three CVSS 10.0 fleet-wide RCEs, while also warning that an unconstrained agent modified a live target and bricked it [record_id:2629]. A Chrome/Android record claims “feedback-driven verification” harvested 100+ zero-days in two months and chained seven high-severity flaws into an Android kill chain [record_id:2628].

AI-assisted workflows also appear in defensive and analytical contexts. Ransomware recovery work used LLMs not to discover the core cryptographic flaw, but to audit state-machine logic and generate verification scripts, accelerating a no-ransom recovery and high-performance decryptor engineering [record_id:2616]. Vulnerability classification research evaluates agentic reasoning and supervised-learning approaches over findings from 15+ tools, shifting AI use from discovery to triage and ranking [record_id:2653]. ThreatForest uses six specialized agents plus deterministic verifiers and ATTACK-BERT mapping to generate attack trees from source code, explicitly criticizing one-shot prompt wrappers [record_id:2664]. Agentic detection engineering at Roblox uses multi-agent pipelines and LangGraph state machines with bounded creativity, tree-of-thought reasoning, and isolated trust boundaries [record_id:2623].

A recurring lesson is that AI appears most useful when embedded in a disciplined verification loop. The records repeatedly contrast bounded decomposition, feedback, deterministic verification, and tool integration with unconstrained agent autonomy [record_id:2619] [record_id:2623] [record_id:2628] [record_id:2629] [record_id:2664].

### 3. Trust-boundary failures dominate cloud, identity, and multi-tenant systems

Cloud and enterprise identity records repeatedly show that isolation breaks not only through memory corruption but through delegated authority, confused deputy patterns, mis-scoped managed identities, and invisible machine credentials.

Non-human identities are a central concern. One record states that the NHI-to-human identity ratio reached 144:1 in 2026 and argues that service accounts, API keys, OAuth apps, CI/CD secrets, Kubernetes identities, and AI agent credentials are persistent, fragmented, and poorly governed. It introduces NHI-Hound to discover and graph machine trust relationships [record_id:2620]. Pass-the-Passkey extends traditional lateral-movement thinking to WebAuthn/passkeys, claiming vulnerable implementations allowed impersonation of privileged users while bypassing phishing-resistant MFA and evading popular XDR tools [record_id:2590]. Active Directory remains a major theme: KerberLoss and ResetNightmare are presented as identity confusion flaws causing Kerberos downgrade, DoS, and full domain takeover [record_id:2647].

Managed cloud identity is another recurring pattern. Azure Automation flaws are described as enabling cross-tenant managed identity takeover, including compromise of sensitive resources and internal Microsoft environments [record_id:2672]. A managed identity chains talk generalizes confused deputy vulnerabilities across Azure and GCP, arguing that low-privileged users can escalate through platform services that hold excessive privileges [record_id:2684]. A major cloud database service record describes escaping a custom .NET sandbox, moving laterally, and extracting a master key that granted admin access to every customer database on the platform [record_id:2668]. Multi-cloud incident response shows the operational side: attackers moved from help-desk phishing to identity provider access, GitHub, and AWS production environments while executing malicious cryptocurrency transactions [record_id:2599].

Kubernetes and container records make similar points about isolation assumptions. One record says namespace-based multi-tenancy is hard and shows vulnerabilities in Kubeflow, Istio, and Traefik that violate workload boundaries [record_id:2679]. Another exposes limitations in seccomp-style filtering for microservices and proposes stateful syscall defense using sequential enforcement, LSM-BPF inline blocking, and OCI-hook policy injection [record_id:2617]. A Linux networking logic bug record claims a path from unprivileged local user to root on Ubuntu 24.04 and from a default Kubernetes pod to managed cloud node compromise [record_id:2678].

### 4. Supply chain risk moves below packages and above repositories

The supply-chain records broaden the target from dependency packages to every layer that executes, builds, scans, updates, or distributes code.

Security scanners themselves are presented as supply-chain weapons: one record describes submitting malicious repositories to 20 scanner vendors and compromising five via features such as external-checks-dir, gemspec evaluation, setup.py execution, and symlink traversal, arguing the broken assumption is that repository analysis is read-only [record_id:2625]. GitHub telemetry is framed as an EDR source for the development platform, with behavioral anomaly scoring over platform events and Git object inspection to detect tag poisoning, workflow abuse, credential-driven takeover, release tampering, commit forgery, and AI workflow prompt injection [record_id:2675].

Enterprise update infrastructure is another target. WSUS is described as enabling organization-wide implant deployment if compromised, with a new attack path for full WSUS infrastructure takeover and malicious update packages for domain-wide code execution [record_id:2613]. Language release pipelines push the problem deeper: vulnerabilities in authentication, credential handling, integrity enforcement, and build isolation behind Go, Python, Flutter, and other release pipelines allegedly enabled backdoored official binaries signed, checksummed, and served from official CDNs [record_id:2673]. ATM supply-chain research targets CryptWare CryptoPro Secure Disk for BitLocker as a proprietary wrapper in Diebold Nixdorf’s Vynamic Security Suite, claiming nine CVEs and master BitLocker key recovery via hidden data blocks, TPM sealing logic, and proprietary AES256 deconstruction [record_id:2598].

Developer workstations and AI agent skill ecosystems represent newer supply-chain edges. One empirical study of attacker C2 infrastructure tied to malicious npm packages and fake job interviews reports roughly 96,000 infected developer workstations, active credentials across 175+ organizations, and widespread production database, cloud, repository, and contractor-related credential exposure [record_id:2660]. Promptware EOD describes malicious AI-agent skills in marketplaces, proposes an agent detonation chamber based on kernel-level observations rather than LLM judges, and reports hundreds of malicious skills among tens of thousands detonated [record_id:2669].

### 5. Legacy and embedded systems remain fertile because old assumptions persist

The records repeatedly challenge the idea that only new technology deserves attention. Legacy Linux services, industrial protocols, BMCs, EOD robots, tractors, GPS trackers, and GPON deployments show long-lived systems with outdated security models and high physical impact.

One record explicitly asks whether attackers need modern AI/cloud attacks when legacy services still expose unauthenticated RCE and LPE. It describes GNU-TelnetD shell injection, TelnetD privilege escalation, and Samba RCEs CVE-2026-4480 and CVE-2026-4408, warning about routers and printers that organizations forget to update [record_id:2585]. BMC research revisits IPMI and finds that three out of four internet-facing IPMI hosts still offer valid usernames and password hashes to network attackers, with default/common internal passwords and crackable factory-randomized credentials [record_id:2670].

Cyber-physical records emphasize safety consequences. Tractor brake-controller research says a recall attributed to power-line noise also removed vulnerable parsing and interrupt-handling code, with DoS and RCE on brake controllers validated on a bench and in a moving truck [record_id:2609]. EOD robot research maps attack surface across operator control units, radio links, and PackBot systems that rely on legacy Linux and Python 2.5, culminating in full compromise of a deployed system [record_id:2596]. Medical-device work proposes a seven-phase Medical Device Kill Chain to model progression from debug ports and firmware extraction to protocol/backend trust weaknesses and patient-impacting outcomes [record_id:2649].

IoT supply chains are repeatedly shown as systemic. TP-Link Omada ZTP research reports 17 vulnerabilities, hard-coded cryptographic keys, information disclosure, RCE, and impacts across Omada, Tapo, Kasa, Festa, VIGI, cameras, mobile apps, and cloud accounts [record_id:2592]. GPS tracker research claims 45 vulnerabilities across SETracker, SinoTrack, and TKSTAR/Thinkrace ecosystems, including device and backend RCE, wiretapping, covert video, vehicle tracking, door unlock, fuel cutoff, and white-label backend concentration [record_id:2646]. Ubiquiti AirMax research reports over-the-air unauthenticated kernel RCE from kilometers away across many device families, challenging “adjacent” severity assumptions [record_id:2624].

### 6. Protocol and parsing mismatches are still a rich attack class

Application and network protocol records highlight parsing and semantic gaps across HTTP, Unicode, CSS/HTML email, TCP timestamps, QPACK, RPKI, AFC, GPON, and compiler transformations.

HTTP desync appears in multiple forms. CRLF-powered desync uses header injection to enable request smuggling even against RFC-compliant proxy chains, with impacts including account takeover, cache poisoning, response queue poisoning, access-control bypass, and “desync worm” potential [record_id:2587]. HTTP Terminator extends desync research via autonomous AI discovery [record_id:2594]. HTTP/3 race research argues attackers can deterministically orchestrate server-side timing via QPACK head-of-line blocking, dynamic table saturation, and priorities, reporting 96.4% execution precision and an 87% vulnerability rate among 10,000 top domains [record_id:2651].

Unicode and CSS records show client-side/application-layer trust failures. Unicode research frames modern Unicode as a distributed parsing vulnerability class across URL decoding, UTF-8 validation, WAFs, frameworks, databases, browser canonicalization, and LLM preprocessing, with examples including illegal UTF-8, surrogate replacement, hex overflows, cookie protection bypasses, WAF/browser XSS/RCE mismatches, invisible variation selectors for LLM jailbreaks, and concealed supply-chain malware [record_id:2604]. CSS-in-email research claims account takeover on major email providers using only CSS and HTML, bypassing sanitization, CSP, and trusted filtering libraries [record_id:2595].

Network timing and routing records show protocol-level side channels and validation gaps. TCP timestamp attacks improve remote timing attacks, enabling a transatlantic Lucky13 attack and jitter-resistant SSH/FTP user enumeration [record_id:2626]. RPKI fuzzing found 21 vulnerabilities across major vendors by abandoning sequential fuzzing for batch-aware, cryptographically coherent input generation and instrumented execution steering [record_id:2671]. Wi-Fi AFC research finds that standard-power 6 GHz access points trust geolocation, DNS, NTP, and TLS assumptions in ways that allow location spoofing, persistent DoS, response injection, and service exhaustion [record_id:2676]. GPON research argues that the threat model has not kept pace with fused ONT/router deployments and 4G/5G backhaul sharing, enabling OLT compromise and subscriber-home pivots via firmware updates [record_id:2663].

### 7. Malware and threat intelligence increasingly target ecosystems and suppliers, not just campaigns

Threat-intelligence records focus on infrastructure visibility, supplier ecosystems, and operational workflows.

A North Korean IT worker record describes forensic images from VPS systems used by operators, revealing cryptocurrency mining, freelance work, corporate reconnaissance, infrastructure supporting fraudulent employment, AI adoption, and detection strategies [record_id:2586]. APT supplier research describes an OPSEC failure by a malware supplier using a public service as a dead-drop resolver, preserving commands, screenshots, and files that exposed multiple APT groups and a managed “Evasion-as-a-Service” business with generative AI integrated into malware development workflows [record_id:2591].

LightSpy research maps 72 C2 servers, live Vue.js panels, cross-platform plugins, router implants, destructive iOS plugins, and operator workflows [record_id:2659]. LANDFALL research follows malformed DNG files uploaded to VirusTotal into a state-linked Android spyware framework exploiting a Samsung image-processing zero-day, likely delivered via WhatsApp, and connects Android infrastructure to Windows intrusions in the Middle East [record_id:2639]. iOS exploit research frames 2026 as pivotal because Coruna and DarkSword watering-hole attacks impacted hundreds of millions of iPhones and proliferated after leaks [record_id:2681]. V8 JavaScript bytecode malware work develops static deobfuscation for JSCeal, a cryptocurrency stealer using compiled bytecode, Brotli compression, Node.js runtime bundling, and layered obfuscation [record_id:2627].

Cybercrime policy and takedown records broaden from technical indicators to disruption strategy. One policy talk argues ransomware networks and pig-butchering scam compounds are converging into gray-zone transnational criminal ecosystems and proposes a six-phase disruption framework [record_id:2674]. LockBit takedown leaders describe Operation Cronos as a multi-phase campaign involving infrastructure seizure, source-code and admin-panel access, affiliate mapping, cryptocurrency account freezes, decryption-key recovery, identification of LockBitSupp, indictments, sanctions, arrests, and sustained credibility destruction [record_id:2685]. A related legal-policy talk examines letters of marque, hack-back, private-sector offensive action, and AI acceleration through constitutional, statutory, operational, and historical lenses [record_id:2682].

## Methods, Tools, And Approaches Discussed

The records describe a wide range of methods, many intended for release as open-source tools or frameworks.

**Reverse engineering and binary analysis** are prominent. Talks reverse engineer Android hypervisors, PackBot robots, tractor brake-controller firmware, Ubiquiti AirMax/AirOS, fingerprint sensor protocols, V8 bytecode malware, macOS GCD/XPC behavior, BTR.sys, RPKI validators, compiler pipelines, and cloud runtimes [record_id:2584] [record_id:2596] [record_id:2609] [record_id:2611] [record_id:2624] [record_id:2627] [record_id:2635] [record_id:2654] [record_id:2671] [record_id:2683]. Tooling examples include IDA Pro scripts and QBinDiff configurations for S12X firmware analysis [record_id:2609], BTR_CLI for Defender remediation-driver transactions [record_id:2611], View8 extensions and deobfuscation filters for V8 bytecode [record_id:2627], and AirMax recon/analysis tooling [record_id:2624].

**Fuzzing and rehosting** appear in embedded and cryptographic contexts. Tesla Wall Connector research uses rehosting-based firmware fuzzing to run bare-metal firmware in an auto-generated environment, finding RCE in UDS-over-CAN update logic and secure boot bypasses [record_id:2656]. RPKI fuzzing introduces batch-aware, cryptographically coherent fuzzing with execution side-channel steering and syntax-tree abstractions [record_id:2671]. These records suggest fuzzing is moving beyond file parsers toward stateful, multi-object, cryptographically constrained, and bare-metal targets.

**LLM-assisted workflows** are diverse. HTTP Terminator is described as an autonomous research cascade for HTTP desync [record_id:2594]. Prompt2Own uses an LLM kernel agent for PoC construction, bug-trigger optimization, primitive discovery, and primitive composition [record_id:2612]. SiftRank uses LLM batch ranking instead of full-agent autonomy [record_id:2619]. ThreatForest uses six agents plus deterministic verifiers to generate attack trees from source code [record_id:2664]. Agentic detection frameworks use specialized agents, Tree-of-Thought reasoning, LangGraph state machines, and runtime validation [record_id:2623]. Closed Loop combines autonomous exploitation with automated Falco, WAF, Sigma, and Terraform defensive outputs [record_id:2658].

**Cloud and identity mapping** methods include graphing non-human identities and trust relationships with NHI-Hound [record_id:2620], analyzing Azure/GCP confused deputy chains [record_id:2684], assessing Teleport clusters and compromised-node post-exploitation paths [record_id:2615], and investigating multi-cloud attack timelines across IdP, GitHub, and AWS [record_id:2599]. GitHub Threat Detector applies EDR-style behavioral baselining to GitHub event streams and Git object inspection [record_id:2675].

**Detection engineering** methods recur across email, endpoint, macOS, GitHub, AI, and enterprise deception. Detection Beyond the Inbox extends email telemetry into SIEM, providing Sigma rules for display-name mismatch, new keyword domains, authentication failures, credential harvesting, and temporal anomalies [record_id:2588]. macOS GCD/XPC work translates queue pressure, starvation, QoS blocking, and XPC execution patterns into detection analytics [record_id:2635]. LOTLE uses environment-specific decoys, tokenized documents, OpenCanary, Caldera, Engage, Blue Agave, and LLM-generated believable artifacts [record_id:2657]. GAVEL proposes rules over neural activations [record_id:2652]. AI investigator benchmarks pit red and blue agents against each other in production-like Active Directory environments [record_id:2666].

**Architectural defenses** include module-boundary control-flow restrictions with PLaTypus on top of Intel CET [record_id:2634], stateful syscall defense using LSM-BPF and OCI hooks [record_id:2617], deterministic-compatible Layer-2 cryptographic protection for TSN cyclic traffic [record_id:2593], hardening recommendations for AFC certificate pinning/geofencing/secure protocols [record_id:2676], sandboxing architecture for Claude Code at Roblox [record_id:2655], and privacy-rights orchestration infrastructure at Roblox using federated service ownership, metadata catalogs, and workflow orchestration [record_id:2637].

## Notable Talks, Records, And Evidence

Several records stand out as especially representative or high-impact within the corpus.

**AI browser and agent compromise records** are among the clearest signals of the conference’s direction. The AI browser record reports universal vulnerability among analyzed AI-powered browsers to indirect prompt injection and emphasizes architectural guardrails [record_id:2583]. PleaseFix escalates the same concern into zero-click takeover of agentic browsers, account takeover across major services, data exfiltration, persistence, and local RCE [record_id:2667]. The retail AI assistant record is notable because the attack occurs through a public mobile interface and claims backend tool execution without infrastructure compromise [record_id:2641]. The official-workflows and agent-framework records are important because they generalize the issue from individual applications to recurring trust-handoff and data-plane/control-plane failures across major AI coding and agent frameworks [record_id:2644] [record_id:2648].

**Cloud multi-tenancy and identity records** are notable for their blast radius.
