## Executive Summary

The CAMLIS source is a multi-year collection of conference talk abstracts and listings centered on applied machine learning for information security. Across 2017–2025, it shows a clear evolution: early records emphasize malware classification, endpoint and network detection, feature engineering, clustering, and scalable security data pipelines; later records increasingly focus on large language models, agentic AI, prompt injection, AI red teaming, privacy, and autonomous cyber defense. The collection is not a single research program but a conference-shaped landscape: applied industrial talks, academic-style papers, keynotes, tutorials, demos, datasets, and some sparse or empty program entries.

## Major Themes And Trends

### 1. From malware ML toward broader security ML—and then toward GenAI security

The 2017–2019 portion of the source is dominated by malware and detection-oriented ML. Talks address raw-byte representation learning [record_id:262], semi-supervised file representations [record_id:239], malware embeddings such as Worm2Vec [record_id:233], graph representations of malware relationships [record_id:217], malware tagging [record_id:219], malware visualization and explainability [record_id:218], adversarial malware examples [record_id:213], static-analysis robustness [record_id:220], backdoor poisoning in malware classifiers [record_id:227], and EMBER dataset improvements [record_id:226]. This period also includes many SOC and network analytics talks: anticipatory defense [record_id:232], adversary simulation as labeled data [record_id:237], hierarchical enterprise network classification [record_id:257], and low-volume DDoS impact modeling [record_id:258].

By 2021–2022, the source expands from “model builds detector” to “model supports a security workflow.” Examples include alert clustering to reduce SOC workload [record_id:192], attack campaign extraction [record_id:196], incident report summarization and visualization [record_id:197], prioritization of incident reports from multiple sources [record_id:199], weak-signal model evaluation [record_id:184], and SQL-driven MLOps infrastructure [record_id:157]. Malware remains important, but records increasingly emphasize dataset quality, model confidence, open-set recognition, and operational deployment [record_id:195] [record_id:202] [record_id:206].

From 2023 onward, generative AI becomes a major center of gravity. The source includes prompt injection via repeated sequences [record_id:170], prompt-injection tutorials and defenses [record_id:171] [record_id:144], LLM model extraction [record_id:167], LLM cyber capability benchmarking [record_id:168], LLM agents for CVE verification [record_id:138], LLM-based threat attribution [record_id:137], model backdoors and Trojan mitigations [record_id:150], privacy-preserving LLM inference [record_id:149], and AI red-teaming frameworks [record_id:135]. The 2025 records intensify this shift toward agentic AI, with privacy in agentic systems [record_id:111], multi-turn jailbreak frameworks [record_id:112], defenses such as LLM salting [record_id:113], graph-level LLM backdoors [record_id:114], VLM prompt-injection evaluation [record_id:115], and AI red-team sessions on computer-use agents, PyRIT, BlackIce, scam agents, package hallucinations, and red-teaming AI red-teaming [record_id:127] [record_id:128] [record_id:129] [record_id:131] [record_id:132] [record_id:133].

### 2. Security ML is treated as an adversarial systems problem, not just a prediction problem

Many records emphasize that security ML models are attacked, drift, or fail under deployment constraints. Malware adversarial examples are a recurring concern, including reinforcement-learning attempts to make Meterpreter evasive [record_id:213], overlay-byte evasion and regularization defenses [record_id:220], backdoor poisoning through crowdsourced threat feeds [record_id:227], ransomware behavior detector red-teaming [record_id:230], and adversarial robustness for tree-based classifiers [record_id:249]. Later work generalizes adversarial ML beyond malware: phishing webpage detectors are attacked with query-efficient adversarial HTML manipulations [record_id:165], client-side perceptual hashing is shown vulnerable to detection-avoidance attacks [record_id:212], deep trading policies are attacked [record_id:203], and XAI explanations themselves are framed as an attack surface for privacy and model compromise [record_id:204].

For LLMs, adversarial concerns shift toward prompt injection, jailbreaks, backdoors, and model extraction. Spotlighting is proposed as a defense against indirect prompt injection [record_id:144]. Residual stream activation analysis is used for attack prompt classification [record_id:148]. Model Leeching extracts task-specific capability from ChatGPT-3.5-Turbo at low API cost and uses the extracted model for attack staging [record_id:167]. In 2025, HarmNet targets adaptive multi-turn jailbreaks [record_id:112], LLM salting rotates internal refusal directions to reduce prompt reuse effectiveness [record_id:113], ShadowLogic introduces graph-level backdoors in white-box LLMs [record_id:114], and Text2VLM evaluates typographic prompt injection against visual language models [record_id:115].

The trend is a move from “ML as detector” to “ML as contested infrastructure.” Models are not merely tools; they are assets, targets, and sometimes agents that create new attack surfaces.

### 3. Graphs, embeddings, and representation learning recur across domains

Graph and embedding methods appear throughout the source. Early malware graph work treats malware, IPs, URLs, and domains as a heterogeneous graph and applies DeepWalk or GraphSAGE-style embeddings [record_id:217]. Later records use graph-based analytics for insider-threat detection via bipartite user-resource graphs and matrix factorization [record_id:154], Kubernetes threat detection via graph neural network embeddings of session events [record_id:153], heterogeneous relational graph convolution for Azure sign-in detection [record_id:175], threat-graph embeddings for personalized threat intelligence [record_id:145], and GraphRAG-inspired cloud infrastructure reasoning [record_id:118]. OmnibusCyber proposes a strongly typed TypeDB-based schema for cybersecurity knowledge graphs and inference [record_id:189].

Embedding and representation learning are also central to malware and artifact analysis. Worm2Vec embeds opcode and operand patterns [record_id:233], metric embeddings streamline malware analysis pipelines [record_id:172], byte-sequence autoencoders learn features without PE-specific preprocessing [record_id:176], FASER uses intermediate representations and long-document transformers for cross-architecture function search [record_id:155], and structure/semantics-aware malware classifiers inject PE section information into vision-transformer-style models [record_id:141]. Threat intelligence work uses embeddings for personalization and retrieval [record_id:145], and cyber campaign detection vectorizes heterogeneous logs for pairwise comparison [record_id:193].

### 4. Analyst workflow, explainability, and human-in-the-loop systems are persistent concerns

A repeated theme is that security ML must fit analyst workflows. Alert clustering is presented as a way to reduce false positives and let analysts reason at the cluster level [record_id:192]. Incident-report summarization and visualization reduce report size and improve precision while preserving narrative structure [record_id:197]. Automated prioritization of incident reports uses self-attention for explainability over multimodal report features [record_id:199]. Threat-prediction visualization for SOC analysts shows model predictions, decision paths, and salient features while allowing analyst feedback [record_id:244]. Insider-threat NLP visualization supports investigators asking narrative questions about motivation, assets, controls, and bypasses [record_id:201].

Explainability also appears in malware and model-security work. CNN malware heatmaps over disassembled images are proposed for reverse engineers [record_id:218]. Activation analysis of byte-based malware classifiers bridges model activations with PE semantics [record_id:238]. Integrated gradients are used to validate ransomware behavior classification and understand adversarial bypasses [record_id:230]. Residual activation analysis is used for LLM attack detection [record_id:148]. A keynote emphasizes shape constraints and rate constraints to control model behavior [record_id:242]. The source consistently frames explainability as operational: not merely interpretability for data scientists, but trust, triage, and safe deployment for defenders.

### 5. Datasets, labeling, and evaluation are treated as core security problems

The source contains unusually strong attention to data. SOREL-20M offers nearly 20 million files, metadata, labels, vendor detections, tags, disarmed samples, and baselines [record_id:195]. BETH provides over eight million data points from 23 hosts for anomaly and out-of-distribution research [record_id:194]. PEVuln creates a four-class PE malware vulnerability dataset using Malvuln and ExploitDB [record_id:140]. MalDICT releases datasets for malware behavior, platform, exploitation, and packer classification, using ClarAVy to parse 882 AV label formats across 90 AV products [record_id:160]. Wild C is introduced as a representative C/C++ corpus for analyzing vulnerability-detection datasets and discovering duplication and data leakage [record_id:198]. EMBER improvements focus on harder test-set selection, feature corrections, duplication reduction, and versioning [record_id:226].

Label noise and weak supervision are recurring. Malware labeling over time using VirusTotal reports addresses temporal label dynamics and rescan prioritization [record_id:240]. Scalable malware labeling infrastructure emphasizes delayed, changing, incomplete labels and rollback requirements [record_id:214]. StringSifter uses Snorkel labeling functions to rank malware strings [record_id:229]. Firenze evaluates models using weak signals and domain-expert “markers” when labels are noisy or biased [record_id:184]. MADAR addresses continual malware learning and catastrophic forgetting with diversity-aware replay [record_id:121]. Small effect sizes in malware detection are addressed by constructing harder train/test splits from malware family information [record_id:158]. Cost-aware evaluation is another subtheme: Cscore replaces F1 for settings where false positives and false negatives have unequal costs [record_id:142], and earlier work proposes cost-cluster auxiliary losses for asymmetric content-classification costs [record_id:245].

### 6. Autonomous and agentic cyber defense develops in parallel with AI-agent risk

Autonomous cyber defense appears early as a keynote vision for host-based “bonware” agents using ML to parry malware [record_id:231]. Later work applies reinforcement learning to network defense, exfiltration-path analysis, operational technology, and maritime systems [record_id:162] [record_id:163] [record_id:185]. Co-Decyber decomposes multi-context OT defense decisions into smaller decisions for multiple agents and applies it to GPS spoofing in vehicle platooning [record_id:136]. 2025 records continue with TERLA for cyber defense agents generalizing across network topologies [record_id:116], contextual reinforcement learning for mission-ready cyber defense policies that adapt without retraining [record_id:120], and test-time compute for real-world cybersecurity agentic systems such as container vulnerability analysis and server alert triage [record_id:117].

At the same time, agentic AI is framed as a new risk surface. The privacy keynote describes autonomous agents moving data through tools, APIs, modalities, persistent memory, workflows, and inference systems, creating new data sinks and leak paths [record_id:111]. The 2025 CAMLIS RED entries suggest practical red-teaming attention to computer-use agents, adversarial simulation, blue teaming, scam-call simulation, package hallucination, and AI security testing tooling [record_id:127] [record_id:129] [record_id:130] [record_id:131] [record_id:132].

## Notable Talks, Records, And Evidence

Several records are especially representative of the source’s technical depth and evolution.

- **SOREL-20M** is a major dataset record: nearly 20 million files, pre-extracted features, metadata, high-quality labels, vendor detection information, tags, disarmed samples, and baseline models [record_id:195]. It exemplifies CAMLIS’s emphasis on reproducible security ML resources.
- **EMBER Improvements** gives a careful account of dataset evolution, harder test-set construction, feature corrections, duplication reduction, and versioning [record_id:226]. Together with SOREL-20M [record_id:195], it anchors the malware-dataset lineage.
- **BETH Dataset** provides real cybersecurity anomaly-detection data with more than eight million data points across 23 hosts and a clean benign/attack host structure [record_id:194]. It is a strong representative of endpoint anomaly-detection dataset work.
- **Spotlighting for indirect prompt injection** provides one of the clearest empirical LLM-security claims: attack success reduced from greater than 50% to below 2% in experiments with GPT-family models, with minimal task impact [record_id:144].
- **PyRIT** introduces an open-source, composable, model- and platform-agnostic GenAI red-teaming framework for multimodal models [record_id:135]. Its recurrence in a 2025 CAMLIS RED title on accelerating AI red teaming suggests continuity and practical uptake [record_id:128].
- **Curl** is a technically detailed privacy-preserving LLM inference record, claiming wavelet-encoded lookup tables improve MPC approximation and reduce rounds/communication versus CrypTen, Iron, and Bolt [record_id:149].
- **LLM Agents for CVE verification** is a representative bridge between LLMs and operational vulnerability management: a planner creates context-sensitive checklists, a RAG/tool-using agent executes them, and summarizer/categorizer LLMs produce VEX justifications [record_id:138].
- **Graph-based insider threat detection** is a strong example of graph ML applied at enterprise scale, reducing raw audit log interactions into a heterogeneous weighted bipartite graph and link-prediction framework [record_id:154].
- **MFA bypass phishing detection** is a substantial applied identity-security record, using point-in-time DNS, targeted filtering, and labeled malicious authentication data to outperform unsupervised methods [record_id:156].
- **Activation analysis of byte-based malware classifiers** is notable for connecting deep model activations to executable semantics and reverse-engineer validation over more than 15 million executables [record_id:238].
- **Backdoor poisoning against malware classifiers** is a key adversarial ML record because it adapts backdoor poisoning to malware and crowdsourced threat-feed settings using SHAP-guided feature manipulation [record_id:227].
- **Agentic AI privacy keynote** is the clearest strategic framing of 2025: agents create new data sinks and leak paths via tools, APIs, memories, workflows, and multimodal contexts, requiring data minimization, friction, computational offloading, TEEs, and new frameworks [record_id:111].

## Gaps, Limits, And Open Questions

The source is rich but uneven. Many records are abstracts with claims but no full experimental details, datasets, or code availability in the raw text. Some provide strong quantitative results, such as SOREL-20M baselines [record_id:195], DIP-ECOD AUC comparisons [record_id:139], Cscore cost savings [record_id:142], prompt-injection spotlighting reductions [record_id:144], Curl performance improvements [record_id:149], and MARL OT-defense outcomes [record_id:163]. Others provide only broad descriptions or titles, especially keynotes and CAMLIS RED entries [record_id:127] [record_id:128] [record_id:131] [record_id:133] [record_id:134] [record_id:143] [record_id:147] [record_id:152] [record_id:161].

Several open questions recur:

1. **Generalization under drift and adversarial pressure.** Many talks identify concept drift, open-set recognition, adversarial examples, poisoning, and changing labels, but the source does not converge on a unified operational standard for measuring robustness across time [record_id:202] [record_id:211] [record_id:227] [record_id:240].
2. **Evaluation realism.** Records repeatedly warn about dataset leakage, unrealistic splits, label noise, and cost-insensitive metrics [record_id:158] [record_id:184] [record_id:198] [record_id:226]. Future research agents should ask whether reported performance reflects deployment conditions.
3. **Human-machine collaboration.** Many systems include analyst interfaces and feedback loops, but the source rarely gives long-term field studies on how analyst behavior, trust, and workload change after deployment [record_id:192] [record_id:197] [record_id:199] [record_id:244].
4. **LLM security measurement.** The GenAI records introduce many attacks and defenses, but there is limited evidence in this corpus about standardized benchmarks for agentic AI red-team outcomes, long-horizon agent safety, or cross-model comparability [record_id:112] [record_id:113] [record_id:114] [record_id:115] [record_id:135].
5. **Privacy in agentic systems.** The 2025 keynote maps future concerns around persistent memories, surveillance, profiling, context stealing, persuasion attacks, ambient AI, and robotics, but concrete evaluation methods for these privacy risks remain underdeveloped in the records [record_id:111].
6. **Governance and ethics.** A few records directly address policy, fairness, or ethical risk—algorithmic bias bounty [record_id:180], fake-account demographic anomaly detection [record_id:247], cost and fairness constraints [record_id:242], geopolitics [record_id:119], and CISA [record_id:134]—but governance is less technically developed than detection and model-security themes.
