# Topic: Source: defcon34

## Executive Summary

The DEF CON 34 source is a broad 2026 conference corpus spanning official main-track talks, village talks, panels, workshops, contests, and community sessions. Collectively, the records describe a security research landscape dominated by five overlapping forces: autonomous and agentic AI as both attack surface and research accelerator; persistent weaknesses in embedded, OT, IoT, maritime, aerospace, medical, and physical systems; renewed exploitation of trusted platform components such as TEEs, TPMs, EDRs, cloud shells, Windows services, and browser/agent sandboxes; large-scale reconnaissance and supply-chain exposure; and the continuing tension between offensive capability, disclosure, governance, and user safety.

The strongest evidence appears in records that describe concrete vulnerability chains, CVEs, released tools, measurements, or live demonstrations. Examples include boot-chain compromise of an Ethereum phone [record_id:2854], Python-in-Excel container privilege escalation [record_id:2855], OMAP-L138 TEE key recovery [record_id:2856], local LLM runtime exploitation in llama.cpp and Ollama [record_id:2858], CPDLC aviation message attacks [record_id:2861], Samsung Internet patch-gap exploitation [record_id:2868], HTTP/2 MadeYouReset research [record_id:2881], automated binary-only 0day discovery [record_id:2882], Kerberos and AD takeover primitives [record_id:2896], GPON wiretapping [record_id:2914], BMC/IPMI fleet exposure [record_id:2918], LiteLLM AI gateway compromise [record_id:2934], TEE physical bus interposition [record_id:2935], and a broad set of AI-agent sandbox, prompt-injection, and agent-benchmarking talks [record_id:2887], [record_id:2925], [record_id:2939], [record_id:2951], [record_id:3084], [record_id:3110], [record_id:3114], [record_id:3117].

The overall trend is that “trusted” infrastructure is repeatedly shown to be trusted too broadly: hardware roots of trust can be emulated [record_id:2915], defensive drivers can become attacker primitives [record_id:2866], EDR installers and Windows protection boundaries can be repurposed [record_id:2870], [record_id:2953], TEEs may protect attested code while leaving deployment state or metadata exposed [record_id:2919], [record_id:3105], and AI agents can perform authorized actions for the wrong semantic reasons [record_id:3102], [record_id:3107]. The corpus also shows that old systems and “boring” protocols remain fertile: telnet, Samba, DICOM, IPMI, GPON, Kannel SMS gateways, NMEA 2000, AIS, and industrial controllers recur as examples where assumptions of closed networks or low attacker interest no longer hold [record_id:2902], [record_id:2906], [record_id:2918], [record_id:2958], [record_id:3019], [record_id:3057], [record_id:3062].

## Major Themes And Trends

### 1. AI and agentic systems are both targets and accelerators

The strongest single thematic current is AI security. Many records reject the idea that AI security is only “prompt injection.” Instead, they treat AI systems as normal software stacks with native code runtimes, sandboxes, file parsers, cloud credentials, MCP tools, browser integrations, model gateways, workflow engines, and supply-chain dependencies.

At the native/runtime layer, local AI systems are framed as ordinary exploitable software: llama.cpp and Ollama are analyzed across JNI, HTTP lifecycle, and Go/C trust boundaries [record_id:2858], while vLLM, PyTorch `torch.load`, ComfyUI, NVIDIA Dynamo, and similar AI infrastructure are described as remotely reachable attack surfaces once “local” model-loading assumptions meet production APIs [record_id:2892]. LiteLLM is described as an AI gateway whose authentication, sandbox, and SSRF weaknesses can chain to cloud compromise [record_id:2934]. Exposed AI infrastructure is also treated as an internet-scale recon target in honeypot and scanning work [record_id:3012], [record_id:3113], and as a resource-hijacking opportunity through exposed inference endpoints and leaked AI API credentials [record_id:3119].

At the agent layer, several records focus on the mismatch between authorization and intent. OpenSearch’s LLM build gate is bypassed not by obvious prompt injection but by exploiting what the model cannot inspect [record_id:2871]. Copilot Studio’s Python environment combines a Python sandbox, a container, and an LLM guardrail, but the record describes bypassing all three and using the guardrail’s leaked reasoning as an oracle [record_id:2925]. Google Gemini CLI is attacked before any prompt is processed, showing that agent startup inputs such as configuration, environment, and protocol state can be dangerous before model-time defenses apply [record_id:2939]. Zero Trust is explicitly challenged for agentic AI because it verifies identity and permissions but not semantic intent [record_id:3102]. Trust amplification in Microsoft Copilot case studies shows AI assistants can add credibility and persistence to attacker-controlled content [record_id:3107]. Agentic browsers are presented as a revived browser-security problem where relaxed browser boundaries and model planning enable zero-click or near-zero-click takeover classes [record_id:3117].

Several records also explore AI as an offensive research engine. HTTP Terminator asks whether autonomous systems can discover new HTTP desync techniques at scale [record_id:2864]. Project Glasswing-style harnessing of frontier models for vulnerability hunting emphasizes that effective AI vulnerability research requires context compression, verification, and false-positive control rather than simply asking a model to find bugs [record_id:3089]. Bug bounty talks repeatedly discuss AI-generated report volume, slop detection, AI-assisted bug hunting, and proof-based multi-agent validation [record_id:2967], [record_id:2992], [record_id:3027], [record_id:3039], [record_id:3044]. Recon talks describe “agents on rails,” hackbots, AI copilots for dark-web and attack-surface mapping, and automated agentic recon pipelines [record_id:3060], [record_id:3068], [record_id:3092]. The corpus is not uniformly optimistic: some records stress cognitive atrophy in OSINT [record_id:2988], operational unreliability of offensive agents [record_id:3036], and the need for AI red teams to separate capability reality from hype [record_id:3069].

AI defenses and evaluation are also represented. AgentBreaker proposes benchmark construction for coding-agent harnesses [record_id:3084]. SADF defines an eight-class taxonomy for agentic security failures and argues that orchestration frameworks themselves are independent attack surfaces [record_id:3110]. Anthropic’s offense-defense boundary talk describes categories of cybersecurity model use and the Cyber Jailbreak Severity framework as an evolving governance mechanism [record_id:3093]. The “abliterated backbone” record tests whether removing refusals makes agents more dangerous and finds that the answer depends on attack class [record_id:3128].

### 2. Trusted computing, roots of trust, and security products are repeatedly inverted

Another major theme is that trusted components become attack primitives. The Ethereum phone talk starts with a bootrom-level misconfiguration and follows compromise into lock-screen verification and wallet-key recovery [record_id:2854]. The OMAP-L138 TEE talk shows a timing side channel in ROM-backed cryptographic module loading that recovers a manufacturer AES key and enables secure-mode compromise [record_id:2856]. Cloud TEE deployment research argues that attestation can remain cryptographically correct while boot-time inputs such as object storage, environment variables, and KMS policies introduce unmeasured compromise paths [record_id:2919]. TEE.fail extends the theme physically, arguing that server-grade confidential VM deployments weakened memory-integrity assumptions enough for DDR5 bus interposition to extract secrets [record_id:2935]. Cove, by contrast, presents a constructive confidential-computing workflow framework using attested DAGs and certificate-gated preconditions [record_id:3088], while another confidential LLM talk emphasizes residual metadata leakage rather than memory compromise [record_id:3105].

Endpoint security products and Windows trust boundaries are similarly inverted. BTR.sys, a Defender remediation driver, is reverse-engineered into a kernel operation primitive without classic BYOVD exploitation [record_id:2866]. BYOEDR abuses trusted EDR installers to get protected code execution [record_id:2870]. COM and WinRT paths are shown as telemetry blind spots compared with CLI-based detection models [record_id:2936]. MSIX, UWP, AppContainers, package identity, and Windows Runtime are reframed as default-enabled attack surfaces across initial access, post-exploitation, lateral movement, and privilege escalation [record_id:2950]. Additional Windows kernel and LPE records chain Microsoft binaries, WerFaultSecure, System Guard, service logic, file deletion, registry creation/deletion, and performance DLL hijacking [record_id:2953], [record_id:2954]. Apple/macOS trust is also challenged through iCloud Keychain extraction [record_id:2860], GPU-private Metal memory hiding from EDR [record_id:2897], iOS unsigned-code execution without jailbreak via SSH-like process virtualization [record_id:2908], and continued dylib hijacking under modern macOS mitigations [record_id:2912].

Hardware trust is also questioned directly. A $45 FPGA SPI TPM that Windows accepts as a root of trust challenges assumptions around secure boot, measured boot, platform attestation, and hardware identity [record_id:2915]. Thin-client full-disk encryption bypasses show that TPM-backed FDE can fail through boot-chain and vendor implementation flaws [record_id:2905]. ATM security-wrapper research shows a proprietary BitLocker wrapper becoming a backdoor to recover keys [record_id:2910].

### 3. Old protocols, embedded systems, and OT remain critical and underprotected

The records repeatedly show that infrastructure built around implicit trust remains exposed. Aviation CPDLC is described as lacking cryptographic authentication, enabling fake ground stations and message injection against certified avionics [record_id:2861]. 5G baseband parsing is attacked through blind spots in generated ASN.1 decoders and natural-language specification rules [record_id:2929]. GPON fiber deployments are described as observable with low-cost hardware and potentially extensible even when encryption is enabled by attacking upstream OLT assumptions [record_id:2914]. Kannel SMS gateways are framed as critical but historically under-audited telecom infrastructure with trust-model flaws enabling censorship, billing fraud, and audit evasion [record_id:2906]. HTTP/2 stream lifecycle bugs show that protocol-layer resets can leave server work continuing after a stream is considered dead [record_id:2881].

OT and ICS coverage is extensive. Polish energy-sector incident analysis emphasizes lateral movement through forgotten remote access, cellular connectivity, APN security, and assumptions of isolation [record_id:2895], with a complementary electric-sector state-of-attacks analysis [record_id:3009]. Industrial detection and validation records cover hypothesis-based OT hunting [record_id:2980], open-source ICSForge coverage validation across industrial protocols [record_id:2995], OT segmentation as operational change management [record_id:3002], nuclear OT simulation with human-aware multi-agent guardrails [record_id:3024], local LLMs in OT [record_id:3028], and an AI-assisted ICS attack-emulation framework using industrial protocol tools [record_id:3046]. IRONMAP presents internet-scale OT asset discovery, claiming millions of exposed ICS/OT-facing assets and detailed examples such as automatic tank gauges [record_id:3062]. Refrigeration controllers, PACs in critical infrastructure, HMI engineering toolchains, and industrial sensor firmware disclosure round out the OT picture [record_id:3071], [record_id:3081], [record_id:2990], [record_id:3074].

Aerospace and space systems are prominent. Satellite protocol exploitation via SDR and ground-station simulation targets CCSDS/SPP, LoRa/FSK links, CAN bus subsystems, and mission operations centers [record_id:2877]. Aerospace student work covers bounded agentic AI for aircraft cyber defense and a satellite constellation research environment [record_id:2978], Linux process injection against CubeSat flight software and satellite dependency scanning [record_id:2984], open-source onboard satellite software analysis [record_id:3022], AFDX primers [record_id:3005], PX4/MAVLink timing and memory-safety issues [record_id:3042], certification of cybersecurity tools for airborne systems [record_id:3043], NTSB data recovery from damaged aircraft equipment [record_id:3086], and SpaceCOP onboard spacecraft intrusion detection [record_id:3111]. Aerospace policy and workforce material appears in student and career records [record_id:3025], [record_id:3059].

Maritime appears as both infrastructure and domain-specific OT. Subsea cable security is framed as a strategic chokepoint and grey-zone problem [record_id:2966]. Dark Fleet cyber boardings provide an operational Coast Guard cyber perspective [record_id:2956]. Maritime cloud dashboards and vessel automation are presented as internet-to-OT risk [record_id:2994]. Cellular NTN brings terrestrial cellular weaknesses to ships [record_id:3029]. Maritime threat hunting emphasizes asset visibility, third-party connectivity, and NOC/SOC collaboration [record_id:3031]. AIS covert channels [record_id:2958], maritime OSINT [record_id:3055], NMEA 2000 IDS [record_id:3057], and maritime physical/digital telemetry exposure broaden the maritime theme.

Medical and biohacking material covers device safety, privacy, and resilience. Insulin pump threat modeling by newcomers emphasizes safety-critical learning and responsible framing [record_id:2965]. Pacemaker ecosystem reverse engineering [record_id:3011], DICOM abuse as offensive infrastructure [record_id:3019], medical-device resilience tabletop exercises [record_id:3001], patient-harm epidemiology after ransomware [record_id:3051], firmware behavior analysis for medical safety [record_id:3033], fertility analyzer full-chain privacy and authentication failures [record_id:3067], and clinical AI prompt poisoning through PDFs [record_id:3082] collectively treat healthcare security as patient safety, not just data security.

### 4. Identity, access, and social engineering are shifting from credentials to ecosystems

Identity records show attacks beyond simple credential theft. Browser-in-the-Middle architectures weaponize CDP/MHTML or persistent BitM browsers to bypass MFA and harvest post-MFA tokens or maintain operator access [record_id:2859], [record_id:2942]. First-party Microsoft services are abused to send convincing phishing content from trusted domains [record_id:2941]. Passkey research argues that WebAuthn ceremonies may be strong while recovery, synchronization, downgrade paths, and relying-party implementation bugs create practical attack surface [record_id:2945]. Active Directory records include new ADCS primitives, Kerberos downgrade/DoS/full takeover, disabled-object privilege paths, and identity-first adversary emulation [record_id:2885], [record_id:2896], [record_id:2982], [record_id:3017]. Mobile banking and biometric frameworks are framed as “glass perimeters” where RASP, root detection, liveness, and AI deepfake controls fail client-side or pipeline-side [record_id:2884], [record_id:2909].

Social engineering intersects with OSINT and fake personas. North Korean IT-worker infiltration offers an inside view of remote employment fraud and laptop-farm operations [record_id:2923], while related talks discuss spotting fake personas before hiring [record_id:3063] and using Haetae to profile and disrupt North Korean C2 infrastructure [record_id:2975]. Human-risk telemetry is framed offensively as a “Human EDR” that adversaries can model and bypass [record_id:3003]. Fake LinkedIn profile detection uses AI, image forensics, behavior, and graph analysis [record_id:3108]. Physical red teaming records also blend identity, social engineering, and access: practical red-team lessons [record_id:2979], badge cloning [record_id:3038], travel security [record_id:3079], and executive-impact storytelling after physical compromise [record_id:3061].

### 5. Internet-scale measurement and reconnaissance are central

A large subset of the corpus is measurement-driven. WhatsApp contact discovery enumerates billions of accounts and metadata [record_id:2890]. CloudShell abuse reverses AWS/Azure/GCP private protocols and automates distributed workloads and persistence across free CloudShell environments [record_id:2888]. BMC/IPMI work scans internet-facing and internal BMCs, extracts hashes, and demonstrates host-to-BMC and BMC-to-host pivoting [record_id:2918]. GPS tracker research maps 36 million devices across shared Shenzhen supply chains [record_id:2903]. Consumer camera cloud research identifies architectural vendor access and multiple evidence chains across Meari/white-label ecosystems [record_id:2949]. Darkmouse cross-domain recon aligns RIR registration, DNS, service, geolocation, sanctions, and longitudinal snapshots across IPv4 space [record_id:3112]. Cl0p infrastructure tracking emphasizes long-lived hosting and reconnaissance patterns across campaigns [record_id:3050], while infrastructure fingerprinting for threat actors uses JARM, JA3/JA4, SSH host keys, and MurmurHash clustering [record_id:3118]. Webhook hunting, cloud-scale VDP lessons, shadow webhooks, and exposed agents similarly reflect a shift toward asset, event, and ownership mapping rather than isolated bug classes [record_id:2974], [record_id:3014], [record_id:3113].

### 6. Supply-chain boundaries are porous and often misunderstood

Supply-chain research appears across software, hardware, cloud, and underground ecosystems. Underground VPN tools are shown as backdoored supply-chain assets used against other attackers [record_id:2879]. OCI registries are weaponized through malicious registry responses affecting clients, runtimes, and ML engines [record_id:2898]. VS Code extension namespace confusion is presented as dependency confusion for editor extensions, with large measured impact [record_id:2928]. AI orchestration platforms are described as shipping RCE-by-design through workflow trust assumptions [record_id:2927]. LaunchBreak generalizes browser-triggered custom URI exploitation against AI and desktop apps [record_id:2955]. UAiRT shows a physical supply-chain implant in ISP routers via UART and ESP32 [record_id:3045]. EV charging firmware compiler trust, satellite software dependency analysis, and quantum cryptographic codebase analysis extend supply-chain thinking into toolchains and cryptographic dependencies [record_id:3052], [record_id:3022], [record_id:3083].

### 7. Cryptography problems are mostly implementation, deployment, and inventory problems

Cryptography records cover post-quantum migration, implementation bugs, and domain-specific crypto. PQC implementation attacks include memory corruption at deserialization, compiler-induced timing side channels, and ML-DSA fault injection [record_id:2900]. Quantum-readiness audits of codebases and financial cryptographic bill-of-materials work argue that RSA/ECC dependencies are pervasive and poorly inventoried [record_id:3083], [record_id:3090]. Other talks cover real-world cryptographic failures in code, FPE misuse, Russian and Chinese standards, ransomware crypto identification, zero-knowledge proofs, and Web3 signature malleability/replay bug hunting [record_id:3091], [record_id:3116], [record_id:3100], [record_id:3087], [record_id:3053], [record_id:3006]. The common message is not “crypto math is broken,” but that glue code, domain choices, key management, implementation verification, and migration inventory often fail.

## Methods, Tools, And Approaches Discussed

The records collectively emphasize multi-method research. Reverse engineering is pervasive: bootchains and TEEs [record_id:2854], [record_id:2856], DJ software protocols [record_id:2880], Verifone card machines [record_id:2904], heavy truck ECUs [record_id:2943], pacemaker firmware [record_id:3011], eReader firmware [record_id:3049], U-Boot [record_id:3104], and Bluetooth firmware with LLM-assisted waypointing [record_id:2930]. Fuzzing, formal methods, and automated analysis appear in Witchcraft Solver, which lifts stripped binaries to LLVM IR and combines formal verification with directed fuzzing and concolic execution [record_id:2882]. Pyrl applies operational taint analysis to Python class pollution across hundreds of thousands of packages [record_id:2922]. Proton provides agent-guided segmented fuzzing for custom URI exploitation [record_id:2955]. U-Boot fuzzing covers hardware-in-the-loop, emulation, and native binary modes [record_id:3104]. USB Video exploitation and 5G baseband work show coverage-guided fuzzing and spec-derived negative testing [record_id:2946], [record_id:2929].

Internet-scale scanning and measurement are recurring methods: WhatsApp enumeration [record_id:2890], BMC scanning [record_id:2918], GPS tracker platform mapping [record_id:2903], CloudShell protocol automation [record_id:2888], camera cloud audit [record_id:2949], OT asset scanning [record_id:3062], IPv4 cross-domain recon [record_id:3112], AI honeypots [record_id:3012], exposed agent discovery [record_id:3113], and inference endpoint discovery [record_id:3119].

AI-assisted workflows are treated as both subject and method. Some tools use AI to generate exploits or recon hypotheses [record_id:2864], [record_id:2931], [record_id:3068], while others constrain AI through deterministic rails, specialized agents, evidence verification, refusal-filtered scoring, or benchmark construction [record_id:3027], [record_id:3084], [record_id:3092], [record_id:3110]. Agentic frameworks also appear in OT and nuclear simulation [record_id:3024], [record_id:3046], medical IoT prompt-injection threat modeling [record_id:2971], and offensive agent reuse lessons [record_id:3036].

Defensive validation and detection methods include ICSForge PCAP scenario generation [record_id:2995], OT threat hunting [record_id:2980], NMEA 2000 LSTM autoencoder ensembles [record_id:3057], SpaceCOP behavior indicators [record_id:3111], ThreatPatrol CTI/ATT&CK visualization [record_id:3078], SEND narrative deception metrics [record_id:2997], and physical covert-entry forensics [record_id:3004]. Privacy/counter-surveillance tools include ESP32-based detection of cameras, drones, jamming, and robots [record_id:2937], adversarial fabric against facial recognition [record_id:2889], Wi-Fi passive privacy exposure scoring [record_id:3048], and physical tracking beacon analysis [record_id:3034].

## Notable Talks, Records, And Evidence

Several records stand out as representative of the corpus’s strongest evidence.

* **AI infrastructure and agents:** Local runtime exploitation in llama.cpp/Ollama is important because it grounds AI security in memory safety and language-boundary bugs rather than prompt behavior [record_id:2858]. ## Gaps, Limits, And Open Questions

The records are abstracts and descriptions, not full papers in most cases. Evidence strength varies. Some entries cite CVEs, published papers, code releases, large-scale datasets, disclosure status, or named tools, making them stronger for downstream technical analysis [record_id:2881], [record_id:2882], [record_id:2896], [record_id:2918], [record_id:2922], [record_id:2945], [record_id:3057], [record_id:3083], [record_id:3110]. Others are panels, workshops, policy discussions, introductory sessions, or community events with less direct empirical evidence [record_id:2966], [record_id:2969], [record_id:2977], [record_id:3041], [record_id:3070], [record_id:3085], [record_id:3101]. Some abstracts make very large claims—such as “100% bypass,” “every app fell,” “all major systems,” or “billions of accounts”—that require corroboration from papers, demos, responsible disclosure artifacts, or independent replication before being treated as established fact [record_id:2884], [record_id:2889], [record_id:2890], [record_id:2909], [record_id:2949], [record_id:3114].

Several open questions recur:

* **How should agentic AI systems be isolated?** Many records show failure of prompt-time, sandbox, or authorization-only controls, but the corpus has no settled architecture for safe agent containment beyond calls for deterministic boundaries, least privilege, memory/tool separation, and stronger evaluation [record_id:2887], [record_id:2939], [record_id:3102], [record_id:3110], [record_id:3117], [record_id:3129].
* **How can disclosure scale?** Bug bounty and VDP talks repeatedly say AI increases report volume and vulnerability discovery velocity, but triage and remediation workflows lag [record_id:2967], [record_id:2974], [record_id:3030], [record_id:3044].
* **What is the correct safety model for OT and medical AI?** OT agent tools and medical LLM poisoning introduce dual-use and patient-safety hazards, while some records propose simulation, human-in-the-loop gates, or behavior analysis but leave real-world deployment questions open [record_id:2971], [record_id:3024], [record_id:3046], [record_id:3082].
* **How should society respond to surveillance-by-design?** Camera, facial recognition, smart home, GPS tracker, and IoT surveillance records expose systemic access and privacy risk, but long-term governance and consumer protection remain underdeveloped [record_id:2889], [record_id:2920], [record_id:2949], [record_id:3034], [record_id:3065].
* **How mature is post-quantum migration?** Multiple records identify implementation and inventory weaknesses, but the practical migration path for financial systems, open-source projects, and embedded devices remains unsettled [record_id:2900], [record_id:3083], [record_id:3090].
